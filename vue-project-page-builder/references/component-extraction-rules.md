# 组件抽离判断规则

当你判断页面区块是否应该抽成组件时，使用这份规则。

## 倾向抽离的强信号

- 该区块在多个页面中有重复的业务语义
- 该区块自身包含较复杂的交互、校验或局部状态
- 该区块适合形成清晰的 props 与 emits 契约
- 抽离后能明显提升页面文件可读性
- 项目中同层级结构本来就有抽组件先例

## 倾向保留页面内的强信号

- 该区块高度绑定当前页面
- 该区块本身较短，直接放在页面里更清晰
- 抽离后只是挪代码，没有形成更清晰的接口
- 项目里同类型内容通常直接写在页面中

## 抽离时的约束

如果决定抽离：

- 用业务语义命名组件
- props 与 emits 保持小而明确
- 不要预埋过多 slots 或配置项
- 不要围绕单一页面场景搭一层泛化框架
- 新组件内部仍优先复用现有共享组件

## 响应中必须说明的内容

无论抽离还是不抽离，都要结合仓库证据解释原因：

- 如果说抽离符合项目先例，要指出相似页面或相似组件
- 如果说保留在页面中更一致，要指出项目里的页面内实现先例
- 要说明当前是否已经存在复用证据，而不是只基于未来猜测
