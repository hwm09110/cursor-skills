# 项目扫描清单

在现有 Vue 项目中开发页面前，先按这份清单建立上下文。

## 最低扫描范围

- 读取 `package.json`
- 读取路由注册位置和一个相近页面
- 读取一个与目标 UI 相似的共享组件
- 读取相近页面使用的接口请求层
- 读取相近页面使用的 store 或 composable
- 读取相近页面使用的样式写法
- 记录后续要在“开发前检查”里引用的文件路径

## 重点确认项

- 框架与版本：Vue 2 还是 Vue 3
- API 风格：Options API 还是 Composition API
- UI 体系：内部组件还是第三方组件库
- 状态管理：局部 refs、composables、Pinia、Vuex
- 数据流方式：页面直接调接口，还是走 service 层
- 样式方式：scoped SCSS、CSS Modules、工具类、design tokens
- 页面骨架：查询区、工具栏、表格、分页、弹窗、抽屉、Tabs
- 可复用点：相似列表页、表单弹窗、详情卡片、筛选区块

## 重点搜索目标

- 共享搜索表单
- 表格封装和通用列
- 字典、选择器辅助模块
- 上传与文件预览组件
- 详情分区组件
- 抽屉/弹窗表单组件
- 权限包装组件和操作按钮

## 结论冲突时的优先级

优先遵循最近的业务模块，其次遵循共享层，再次才参考框架默认做法。

## 输出提醒

在真正开始编码前，先总结：

- 已识别到哪些项目规范
- 哪些组件或代码可以复用
- 是否需要新建组件
- 这些结论分别有什么代码依据
