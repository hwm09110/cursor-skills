# 缺失信息处理规则

## 基本原则

- 缺失信息不等于允许编造
- 不确定项必须显式记录
- 项目级 skill 只写确认过的规则
- 默认生成到当前仓库内，除非用户明确要求，否则不输出全局安装方案

## 必须写入 `open-questions.md` 的情况

- 无法确认页面主目录
- 无法确认请求库或 request 封装入口
- 无法区分项目级组件和模块级组件
- 无法确认状态管理边界
- 页面模式样本不足
- 多处代码风格冲突且无法判断主流规范

## `open-questions.md` 的写法

每个问题至少包含：

- 问题描述
- 影响范围
- 当前可见证据
- 建议人工补充的内容

## 冲突处理

如果 JSON 和 Markdown 结论冲突：

- 优先保守处理
- 在目标 skill 中不要强写冲突结论
- 把冲突加入 `open-questions.md`
