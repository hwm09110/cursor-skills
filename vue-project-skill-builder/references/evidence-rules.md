# 证据规则

## 证据写法

- 优先使用相对仓库根目录路径。
- 优先引用真实存在的配置、页面、组件、request、store 文件。
- 一个结论最好至少有 1 个证据路径，重要结论最好有 2 个以上。

## 可接受的证据类型

- `package.json`
- 路由配置文件
- 典型页面文件
- 共享组件文件
- request / service / api 文件
- store / composable 文件
- 样式入口与主题文件

## 不确定项处理

如果无法确认：

- 明确写“未确认”或“证据不足”
- 加入 `open_questions`
- 不要把推测写成既定事实

## 禁止事项

- 不要用“通常 Vue 项目会这样”当证据
- 不要只给目录名而没有具体文件路径
- 不要因为找到一个孤立文件就认定它是全项目规范
