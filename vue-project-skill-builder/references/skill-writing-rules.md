# Skill 写作规则

## 生成后的 SKILL.md 应该写什么

- 适用范围
- 开发前需要读取的目录和文件
- 项目内复用优先级
- 页面实现规则
- 组件抽离判断规则
- 输出格式要求
- references 导航

## 生成后的 SKILL.md 不应该写什么

- 大段项目背景介绍
- 与执行无关的解释性文档
- 过长的组件清单
- 完整分析报告原文

## 写法原则

- 先写“必须怎么做”，再写“到哪里查”
- 优先写项目事实与项目先例
- 尽量用中文短句
- 让后续 AI 一眼知道应该先读哪里、再做什么、什么不能做

## References 写法原则

- 用标题清楚归类
- 每节优先给“结论 + 证据来源”
- 方便人工后续直接修改

## 跨编辑器写法原则

即使目标编辑器不同，也统一生成标准 skill 结构：

- `SKILL.md` 承担核心约束
- `references/` 承担详细规范
- `agents/openai.yaml` 保留编辑器可读的基础元数据
- 差异主要体现在目录位置，而不是文件结构
